# 编码规范
## Python 命名规范
- 变量名：snake_case（如 `user_name`）
- 类名：PascalCase（如 `UserProfile`）
- 常量名：UPPER_CASE（如 `MAX_RETRY_COUNT`）
- 私有成员：单下划线前缀 `_private`
- 魔术方法：双下划线包围 `__init__`
- python文件名使用小写字母或者拼音，需要使用连接符的时候，必须使用_连接
## 代码风格
- 缩进：4 个空格（禁止使用 Tab）
- 行长度：最大 88 字符（Black 默认值）
- 导入顺序：标准库 → 第三方库 → 本地模块（每组空行分隔）
- 字符串：优先使用单引号，docstring 使用双引号
## 文档规范
- 每个模块使用模块级 docstring
- 每个公开类/函数必须有 docstring
- docstring 格式：Google Style 或 NumPy Style
## 测试规范
- 测试文件命名：`test_<module_name>.py`
- 测试函数命名：`test_<function_name>_<scenario>`
- 使用 pytest 框架
- 每个函数至少包含：正常路径测试 + 边界测试 + 异常测试

# 程序对配置文件目录读写的设置：
## 默认在程序目录下开始浏览查找并打开文件。
- 优化程序代码，使程序在被pyinstaller或者nuitka打包成exe时，仍然能够指向程序默认目录。
- 分离程序目录和工作目录：
- 程序目录：程序所在的临时目录（用于内部使用）
- 工作目录：用户的工作目录（用于文件对话框）
- 智能获取工作目录：
- 优先使用当前工作目录（如果不在临时目录）
- 其次使用用户文档目录
- 然后使用桌面目录
- 最后使用用户主目录
- 配置文件保存位置：
- 将配置文件保存在程序所在目录中.而不是保存在临时程序目录。
- 文件对话框会正确显示用户的工作目录，而不是临时目录。
- 不要在文件或者程序的打开位置保存程序配置文件。
- 把配置文件的读写保存均固定在程序目录中。
- 程序生成的文件夹,保存在程序的目录下,不要保存在程序或者文件的打开位置。
- 使用 get_safe_output_base() 函数：frozen 时直接用 get_working_dir()（跳过 Nuitka 解压的临时目录）；非 frozen 时用 get_program_dir()（并验证非临时目录）。
- Nuitka 不设 sys.frozen，所以走 else 分支用 __file__ 获取的是临时目录路径。需要增加 Nuitka exe 检测。
- 将 get_program_dir() 改为三重判断：
sys.frozen 为真 → PyInstaller 等打包，用 sys.argv[0] 取 exe 所在目录
sys.argv[0] 以 .exe 结尾 → Nuitka 编译的 exe（Nuitka 不设 sys.frozen），同样取 exe 所在目录
否则 → 普通 Python 脚本，用 __file__
## 临时目录检测：
- 添加 is_temp_directory() 方法检测路径是否是临时目录
- 避免将配置文件保存在临时目录
- 文件对话框初始目录：
- 优先使用上次访问的路径
- 其次使用当前程序所在目录
- 最后使用工作目录
- 程序被nuitka打包成pyd文件，启动pyd文件的exe程序启动程序后，保持程序的配置文件在目录下可读可写
- 查找框可用对文件进行拖放操作
- 如果使用了第三方库，项目完成后生成requirements.txt
